AI Coding Assistants for the Lab

A practical guide to AI tools that help with scripting, data analysis, measurement automation, and lab software — from terminal agents to desktop apps.

What Are AI Coding Assistants?

Whether you're writing Python scripts for data analysis, automating measurement routines, building LabVIEW or EPICS integrations, or just trying to plot XRD data at 5 PM on a Friday — AI coding assistants can help. They range from simple chat interfaces to full autonomous agents that can edit your scripts, run them, and fix errors on their own.

These tools come in three flavours:

  • CLI agents — Terminal-native tools that work directly with your scripts and project files (Claude Code)
  • Cloud agents — Browser-based agents that take your project, work on it in an isolated environment, and deliver the finished changes for your review (OpenAI Codex)
  • IDE extensions — Integrated into your editor with inline suggestions, chat panels, and agentic editing (Claude Code for VS Code, Codex for VS Code, GitHub Copilot, Cursor)
  • Desktop apps — Standalone GUI applications for conversational help, data analysis, and quick questions (Claude Desktop)

You don't need to be a software engineer to benefit. If you write any code at all — Python notebooks, shell scripts, MATLAB routines, data processing pipelines — these tools can save significant time.

Agents vs Standard AI Chats

The key difference: standard chats generate text you copy-paste, while agents work directly on your files — editing scripts, running analysis, and fixing errors autonomously.

Capability Standard Chat (ChatGPT, Claude.ai) Coding Agents (Claude Code, Codex, Copilot)
Access your filesOnly what you paste or uploadFull project access — reads your scripts, data files, configs
Edit scriptsSuggests code you copy-pasteDirectly creates and modifies your .py, .m, .sh files
Run codeNo local executionRuns your scripts, installs packages, executes shell commands
Fix errorsYou copy error messages back manuallySees errors, fixes them, re-runs automatically
Multi-file changesOne snippet at a timeUpdates analysis pipeline + plotting + config together
ContextLimited to conversationEntire project — reads your data formats, existing code, docs
Change trackingNoneAutomatically saves versions of your files so you can undo any change
TestingSuggests test codeWrites tests, runs them, fixes failures
MemoryConversation onlyRemembers project conventions across sessions
SafetyCan't break anything — read-onlyPermission modes let you approve or restrict actions

In short: a standard chat is a knowledgeable colleague you ask questions; an agent is a capable assistant that works directly on your scripts and data pipelines.

CLI Agents

Terminal-native agents that run in your shell. Point them at a folder of analysis scripts, a measurement automation project, or a data pipeline — they'll read everything, make edits, and run your code directly.

Claude Code (CLI)

Anthropic's agentic terminal tool. Lives in your command line, understands your full project, and can edit scripts, run analysis pipelines, install packages, and automatically track changes to your files. Powered by Claude Opus and Sonnet models.

Key Features

  • Reads and understands your entire project — scripts, configs, data formats
  • Edits multiple files at once (e.g. update analysis + plotting + config together)
  • Runs commands directly — pip install, python analysis.py, make
  • Automatic version tracking — saves snapshots of your work so you can always go back to a previous version
  • Memory system — remembers your project conventions across sessions
  • Can connect to external tools and databases (via MCP protocol)
  • Permission modes — control what the agent can do (ask first, auto-edit, fully autonomous)

Strengths

  • Strong reasoning on complex, multi-step tasks
  • Excellent at understanding large collections of scripts
  • Persistent memory across sessions
  • Fine-grained permission and safety controls

Limitations

  • Requires API key or Max subscription
  • Usage costs scale with conversation length
  • Terminal-only (though editor extensions exist)

Cloud Agents

Cloud-based agents that run on remote servers. You give them a task and point them at your project — they copy it, work on it independently in an isolated environment, and present the finished changes for your review. Great for delegating tasks you don't want to babysit.

OpenAI Codex

Codex is OpenAI's cloud-based coding agent, available at chatgpt.com/codex. You describe a task, it creates a private copy of your project in the cloud, works on it (typically 1–30 minutes), and shows you exactly what it changed for your approval. Powered by GPT-5.2-Codex.

Key Features

  • Fully cloud-based — no local setup, runs in an isolated environment
  • Connects to your GitHub projects — reads your code, makes edits, and delivers changes for review
  • Runs multiple tasks in parallel — delegate several jobs at once
  • Executes your scripts and checks for errors in the cloud
  • Real-time progress monitoring — watch what it's doing as it works
  • Internet disabled during execution for security
  • Skills — configurable for code understanding, prototyping, documentation
  • Automations — can pick up routine work automatically

Strengths

  • No local setup — works entirely in the browser
  • Parallel task execution — run many jobs simultaneously
  • Secure — can't affect your local machine
  • Included with ChatGPT Plus, Pro, and Enterprise plans

Limitations

  • Requires your project to be on GitHub — not for local-only files
  • No internet access during execution (libraries must be pre-installed)
  • Not interactive — you assign work and come back to review later
  • Cannot access local lab instruments or data files directly

IDE Extensions

These tools integrate directly into your code editor (VS Code, PyCharm, etc.), providing inline completions, chat panels, and agentic capabilities. Great if you already write scripts in an IDE and want AI help right where you work.

Claude Code for VS Code

The Claude Code VS Code extension embeds the full Claude Code agent directly into your editor. You get a chat panel that can read your project, edit files, run commands, and track changes — all within VS Code.

Key Features

  • Full agent capabilities inside VS Code (same engine as the terminal version)
  • Shows you exactly what it wants to change before applying edits
  • Can run commands from the chat panel (e.g. run your script, install a package)
  • Automatically reads relevant files from your project
  • Remembers project conventions across sessions
  • Also available for JetBrains IDEs (PyCharm, etc.)

VS Code Marketplace · IDE Integration Docs

Codex for VS Code

OpenAI's Codex VS Code extension brings the Codex agent into your editor. You can chat, get inline edits, or delegate tasks to Codex Cloud — all from a sidebar panel. Available on macOS and Linux (Windows experimental).

Key Features

  • Three modes: Chat (conversation), Agent (edits with your approval), Agent Full Access (fully autonomous)
  • Shows proposed changes side-by-side before you accept them
  • Send tasks to Codex Cloud directly from the editor
  • Reads your open files and project structure automatically
  • Also works in Cursor and Windsurf editors

Strengths

  • Seamless bridge between local editing and cloud agent
  • Included with ChatGPT Plus/Pro subscription
  • Three permission tiers for different comfort levels

Limitations

  • Windows support is experimental
  • Newer than Copilot — smaller ecosystem

GitHub Copilot

GitHub Copilot is the most widely adopted AI coding assistant. Originally focused on suggesting code as you type, it has evolved into a full agent that can autonomously edit files, run commands, and fix errors across your project.

Key Features

  • Suggests code as you type — autocomplete on steroids
  • Chat panel for asking questions about your code within the editor
  • Agent mode — autonomously edits files, runs commands, fixes errors
  • Available in VS Code, PyCharm/JetBrains, Neovim, and more
  • Multiple AI models behind the scenes (GPT-4o, Claude Sonnet, Gemini)

Strengths

  • Best-in-class autocomplete suggestions
  • Deeply integrated into the GitHub ecosystem
  • Free tier available
  • Widest editor support

Limitations

  • Agent mode is newer, less mature than dedicated agents
  • Autocomplete suggestions can be distracting when you're thinking
  • Advanced features require paid plans

Cursor

Cursor is a standalone AI-powered code editor (based on VS Code) that puts AI at the centre of everything. Rather than adding AI as a plugin, it's built into every part of the editor — autocomplete, inline editing, chat, and a full agent mode.

Key Features

  • AI-native editor — not a plugin, AI is woven into every interaction
  • Smart autocomplete — multi-line, context-aware suggestions as you type
  • Inline editing — select code, describe what to change in plain English, and it's applied
  • Agent mode — autonomously edits across files, runs commands, installs packages
  • Understands your full project — ask questions about any file
  • Multiple AI models (Claude, GPT-4, and others)
  • Compatible with VS Code extensions — bring your existing setup

Strengths

  • Deepest AI integration of any editor
  • Excellent "select and describe" editing workflow
  • Agent mode for autonomous multi-step tasks
  • Looks and feels like VS Code — easy to switch

Limitations

  • Requires switching from your current editor
  • Subscription required for full features
  • Proprietary software

Desktop Apps

Standalone GUI applications for conversational AI. Great for quick scripting help, analysing data files, understanding error messages, or asking "how do I fit a Gaussian to this spectrum?" — without setting up a terminal or IDE.

Claude Desktop

Anthropic's desktop application for macOS and Windows. A native app for chatting with Claude — great for quick questions, analysing files and images, and getting help with scripts without needing a terminal or editor.

Key Features

  • Native macOS and Windows app with a fast, clean interface
  • Drag-and-drop file and image analysis (CSV files, plots, error screenshots)
  • Built-in screenshot capture and analysis
  • Can connect to local tools and databases (via MCP protocol)
  • Projects with custom instructions and reference files
  • Generates interactive code snippets, documents, and visualisations
  • Global keyboard shortcut for quick access from any app

Strengths

  • Best reasoning of any desktop AI app (Claude Opus/Sonnet)
  • Extensible — connect to lab databases and tools via plugins
  • Excellent for planning analysis approaches and reviewing scripts
  • Analyse screenshots, diagrams, spectra images, and plots

Limitations

  • Cannot directly edit your files (use Claude Code for that)
  • No built-in terminal or command execution
  • Requires Pro/Max subscription for full model access

When to Use What

Use a CLI agent (Claude Code) when:

  • Building or refactoring a multi-file analysis pipeline (data loading + processing + fitting + plotting)
  • Automating measurement routines — e.g. GPIB/VISA scripts, scan sequences
  • Converting MATLAB scripts to Python, or porting between frameworks
  • You want the agent to run your script, see the error, fix it, and re-run automatically
  • Setting up a new project with proper structure, environment, and dependencies

Use a cloud agent (OpenAI Codex) when:

  • You want to delegate a task and come back to review the results later
  • Running multiple independent tasks in parallel (e.g. "add documentation to all scripts" + "write tests for the fitting library")
  • Your scripts are already on GitHub and you prefer a browser-based workflow
  • You don't want to install anything locally

Use an IDE extension (Copilot, Cursor, Claude Code / Codex for VS Code) when:

  • Writing or editing scripts in VS Code or PyCharm and want inline help
  • You want autocomplete that understands your data structures and function signatures
  • Quick edits — "add error bars to this plot", "handle the case where the file is empty"
  • You prefer a visual editor that shows what changed before and after

Use a desktop app (Claude Desktop) when:

  • "How do I fit a Voigt profile to this XPS spectrum in Python?"
  • Drag-and-drop a CSV or screenshot of a plot for analysis help
  • Understanding an error message or unfamiliar library
  • Planning an analysis approach before writing code
  • Asking about physics, statistics, or material science concepts

Use a standard chat (claude.ai, chatgpt.com) when:

  • Quick questions — "what's the syntax for numpy.polyfit?"
  • Learning a new library (pandas, lmfit, matplotlib, scipy)
  • Generating a one-off snippet you'll paste into a notebook

Typical Agent Workflow

Example: "Write a Python script that reads all .dat files in the measurements/ folder, fits each spectrum to a Lorentzian, and saves the fit parameters to a CSV."

  1. Describe the task — Tell the agent what you need in plain language
  2. Agent reads your project — Looks at your existing scripts, data file formats, directory structure
  3. Agent writes code — Creates or modifies scripts, installs packages (pip install lmfit)
  4. Agent runs and iterates — Executes the script, reads errors, fixes issues, re-runs
  5. You review — Check the output, ask for adjustments ("use a Voigt profile instead")
  6. Done — Agent saves your files, optionally creates a version snapshot you can revert to
